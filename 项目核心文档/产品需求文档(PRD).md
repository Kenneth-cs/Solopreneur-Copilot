# 超级个体创业OS (Solopreneur Copilot) 产品需求文档 (PRD)

## 1. 文档综述

### 1.1 项目背景
针对具备大厂PM经验及全栈开发能力的“超级个体”，为了避免陷入“产品自嗨”和“过度追求完美”的误区，需要构建一套自我监督与辅助系统。该系统旨在将工作重心从单纯的“写代码/做功能”转移到“商业验证”与“变现”上来，通过标准化的流程和AI辅助，加速产品矩阵的孵化与验证。

### 1.2 产品定位
*   **核心理念**：以终为始，驱动变现。
*   **产品形态**：Web Dashboard + AI Agent。
*   **核心KPI**：验证速度、离钱的距离。
*   **目标用户**：独立开发者、超级个体、创业者。

### 1.3 核心价值
1.  **基础设施**：提供统一的登录、支付、数据看板，降低新产品边际成本。
2.  **虚拟监工**：通过倒计时和灵魂发问，克服拖延症与完美主义。
3.  **商业参谋**：通过AI市场调研与评分，筛选高价值Idea，过滤伪需求。

## 2. 产品结构与功能模块

### 2.1 功能架构图
*   **Dashboard 首页**：全局概览（MRR、活跃项目、待办事项）。
*   **模块一：创意碎纸机 (Idea Validator)**
*   **模块二：MVP 倒计时锁 (Time-Boxer)**

*   **模块四：矩阵基础设施 (Matrix Core)**
*   **模块五：自动化营销 (Marketing Matrix)**

## 3. 详细功能说明

### 3.1 模块一：创意碎纸机 (The Idea Validator)
**解决痛点**：防止盲目立项，开发伪需求。

*   **3.1.1 Idea 录入**
    *   **输入**：一句话描述产品想法（如“AI写周报工具”）。
    *   **处理**：支持语音或文本快速记录。
*   **3.1.2 AI 市场调研 Agent**
    *   **功能**：后台调用搜索API（Perplexity/Google/Bing）及社交媒体数据。
    *   **分析维度**：
        *   GitHub 趋势与开源竞品。
        *   Product Hunt 热榜情况。
        *   Twitter/小红书/SEO 关键词热度与搜索量。
        *   竞品数量与定价策略。
    *   **输出**：生成一份简短的市场调研报告。
*   **3.1.3 商业画布强制填写**
    *   **规则**：立项前必须回答核心问题（目标客户、痛点、MVP最小功能、变现模式）。
*   **3.1.4 毒舌 VC 评分 (Cold Score)**
    *   **逻辑**：AI扮演刻薄投资人，基于“变现难易度”、“开发成本”、“流量获取难度”打分（0-100分）。
    *   **决策**：
        *   < 60分：建议“扔进碎纸机”，系统拒绝立项。
        *   \> 60分：允许进入下一阶段。

### 3.2 模块二：MVP 倒计时锁 (The Time-Boxer)
**解决痛点**：完美主义拖延，产品迟迟不上线。

*   **3.2.1 立项契约**
    *   **设置**：设定绝对上线日期（建议周期：7天）。
    *   **功能削减**：输入Feature List，AI强制标记70%为“非必要”，只保留MVP核心功能。
*   **3.2.2 倒计时与警报**
    *   **展示**：Dashboard 显著位置显示剩余时间（天/小时/分）。
    *   **红灯机制**：
        *   倒计时结束前未检测到上线/支付链接：触发疯狂报警（邮件/微信通知）。
        *   极端模式（可选）：通过脚本锁定代码编辑器，直到点击“发布”或“放弃”。

### 3.3 模块三：每日复盘 Agent (The Daily Standup)
**解决痛点**：执行力下滑，缺乏每日正向反馈。

*   **3.3.1 定时触发**
    *   **机制**：每晚 10:00 自动弹出对话框或发送通知。
*   **3.3.2 灵魂六问**
    1.  今天进账了吗？
    2.  做了什么推广动作？
    3.  代码的商业价值是什么？
    4.  哪件事离赚钱最近？
    5.  遇到了什么卡点？
    6.  明天如何带来收入？
*   **3.3.3 智能日报**
    *   **输入**：用户的回答 + 当日 Git Commit 记录。
    *   **输出**：生成结构化日报，记录创业心路历程，可作为“Build in Public”的素材。

### 3.4 模块四：矩阵基础设施 (The Matrix Core)
**解决痛点**：重复造轮子，多产品管理混乱。

*   **3.4.1 统一鉴权中心 (Passport)**
    *   **功能**：提供统一的 OAuth/JWT 登录接口，新产品直接接入。
*   **3.4.2 统一支付网关 (Payment Hub)**
    *   **功能**：封装 Stripe/微信支付/LemonSqueezy，统一回调处理。
*   **3.4.3 商业仪表盘**
    *   **数据集成**：聚合所有产品的用户数、日活、收入（MRR/LTV）。
    *   **成本核算**：集成服务器成本，自动计算净利润。

### 3.5 模块五：自动化营销 (Marketing Matrix)
**解决痛点**：重开发轻运营。

*   **3.5.1 营销内容生成**
    *   **触发**：完成重要 Feature 或 Milestone。
    *   **输出**：AI 自动生成适配 Twitter/小红书/即刻 的推文文案，附带产品链接。
*   **3.5.2 流量互导**
    *   **机制**：在基础设施层预埋广告位，A产品可展示B产品的推广信息。

---

## 4. 优先级规划

*   **P0 (第一周)**：
    *   基础框架搭建 (Next.js)。
    *   模块三：每日复盘 Agent (核心逻辑)。
    *   模块一：Idea 录入与基础 AI 评分。
*   **P1 (第二周)**：
    *   模块二：倒计时锁。
    *   模块一：AI 市场调研 (接入搜索API)。
*   **P2 (第三周及以后)**：
    *   模块四：矩阵基础设施 (Passport & Payment)。
    *   模块五：自动化营销。
